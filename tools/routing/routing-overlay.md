### Disambiguation rules (hand-curated — not stated in any agent's own description)

1. Spark imports (`pyspark.*`, `org.apache.spark.*`, `SparkSession`, DataFrame/Dataset ops) → route to the `spark-*` family, never the plain `scala-reviewer`/`python-reviewer`/`scala-implementer`/`python-implementer` pair, even though the file extension matches.
2. Within `spark-*`: checkpoint/watermark/`transformWithState`/streaming-source diffs → `spark-streaming-specialist` first; `spark-scala-reviewer`/`spark-pyspark-reviewer` for everything else in that language.
3. Rust reviewers stack, they don't replace each other: start from `rust-idiom-reviewer` on every diff, and additionally run `rust-performance-profiler` for perf regressions or `rust-api-designer` for public API/semver shape (its own description already routes `unsafe`/FFI to `rust-unsafe-ffi-auditor` and async/Tokio to `rust-async-concurrency-reviewer`). Two more sit outside the per-diff flow: `rust-toolchain-auditor` owns Cargo.toml/lockfile/workspace/CI-gate debt (route dependency or build-config changes there, not to the code reviewers), and `rust-test-strategist` owns whether tests actually prove the change (route "is this tested enough?" questions there).
4. Repo has a `.jj/` directory → `jj-workflow` owns commit/rebase/bookmark operations; don't shell out to raw `git` via Bash for those repos.
5. `architecture-reviewer` complements, not replaces, the language reviewer's line-level pass on a large diff — run architecture-reviewer first, then the language pair.
