# Security Policy

## Supported versions

Security fixes are applied to the latest version on `main`. This project does not
maintain separate release branches.

## Reporting a vulnerability

Do not open a public issue for suspected vulnerabilities or exposed credentials.
Use GitHub's **Report a vulnerability** button in the repository Security tab to
submit a private security advisory.

Include affected files or versions, reproduction steps, impact, and any suggested
mitigation. You should receive an acknowledgement within seven days.
